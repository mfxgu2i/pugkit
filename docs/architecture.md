# pugkit アーキテクチャ方針

## ビルドモードの基本方針

### `build` コマンド（プロダクションビルド）

- **キャッシュを一切使わない** — テンプレートキャッシュ・ハッシュキャッシュは build 時には保存・参照しない
- **フルビルド** — 変更検知はせず、対象ファイルをすべて処理する
- **`clean: true`（デフォルト）** — ビルド開始前に出力ディレクトリ（`outDir`）を全削除してから書き出す
- **`clean: false`** — 既存環境（htdocs 等）に pugkit のファイルを追加・上書きするユースケース向け。出力ディレクトリの既存ファイルは削除しない

> `clean: false` でも内容が変わったファイルだけ書き込むことはしない。必ず全ファイルを書き出す。

### `dev` コマンド（開発サーバー）

- **キャッシュ・依存グラフはプロセスのメモリ上にのみ保持** — ディスクへの永続化は一切しない
- **セッション単位でリセット** — 起動（`pugkit dev`）から終了（`Ctrl+C`）までが1セッション。再起動すると `BuildContext` が新規作成され、キャッシュ・依存グラフはすべて初期状態に戻る
- **ファイル変更時の最小再ビルド** — ウォッチャーが変更ファイルを検知し、影響を受けるファイルのみ再処理する
- **画像変更時は影響する Pug のみ再ビルド** — `imageGraph`（Pug→画像の依存グラフ）をレンダリング時に構築し、変更された画像を参照している Pug ファイルのみ再ビルドする

---

## キャッシュの種類と有効範囲

すべてのキャッシュはメモリ上にのみ存在し、ディスクには保存されない。

| キャッシュ                     | 格納場所                              | dev                                       | build                                    |
| ------------------------------ | ------------------------------------- | ----------------------------------------- | ---------------------------------------- |
| Pug コンパイル済みテンプレート | `CacheManager.compiledCache`          | ✅ セッション中保持                       | ❌ 保存しない                            |
| ファイルハッシュ（変更検知）   | `CacheManager.fileHashes`             | ✅ セッション中保持                       | ❌ 使用しない（全ファイル処理）          |
| 画像バッファ（image-size 用）  | `_imageBufferCache`（モジュール変数） | ✅ セッション中保持（画像変更時に無効化） | ✅ プロセス中保持（1回のビルドで再利用） |

---

## 依存グラフの有効範囲

すべての依存グラフはメモリ上にのみ存在し、ディスクには保存されない。

| グラフ        | 対象                             | dev                               | build                                             |
| ------------- | -------------------------------- | --------------------------------- | ------------------------------------------------- |
| `graph`       | Pug ファイル間の include 依存    | ✅ セッション中保持・随時更新     | ✅ ビルド時に構築（次回 build では再構築）        |
| `sassGraph`   | Sass の `@use` / `@forward` 依存 | ✅ セッション中保持・随時更新     | ✅ 同上                                           |
| `scriptGraph` | JS/TS の import 依存             | ✅ セッション中保持・随時更新     | ✅ 同上                                           |
| `imageGraph`  | Pug ファイル→参照画像の依存      | ✅ レンダリング時に構築・随時更新 | ❌ 構築しない（build は全ファイル処理のため不要） |

> いずれも `BuildContext` のインスタンスプロパティ。`dev` / `build` を再実行するたびに `new BuildContext()` で初期化される。

---

## 出力ディレクトリへの書き込み方針

出力先は `outDir`（デフォルト: `dist`）で変更できる。

| モード                     | 動作                                                 |
| -------------------------- | ---------------------------------------------------- |
| **build** (`clean: true`)  | 出力ディレクトリを削除してから全ファイルを書き出す   |
| **build** (`clean: false`) | 出力ディレクトリを削除せず全ファイルを上書き書き出す |
| **dev**                    | 変更されたファイルのみ再ビルドして書き出す           |

---

## build コマンドの独立性方針

dev サーバーがどんな状態であっても（不具合・キャッシュ汚染・グラフ破損など）、`build` コマンドは常に正しい出力を返せるよう、以下の方針で設計する。

**1. プロセス分離**  
`pugkit build` を実行するたびに新しい Node.js プロセスを起動する。dev サーバーが保持していたすべてのメモリ状態（`_imageBufferCache`・各依存グラフ・コンパイル済みテンプレート）を引き継がない。

**2. production モードではキャッシュを一切参照しない**  
`resolveChangedFiles`（`tasks/pug.mjs`）は `isProduction` のとき全ファイルをそのまま返す。`CacheManager.setPugTemplate` も production では保存しない。Sass・Script のインクリメンタル処理も `isDevelopment` の条件で保護し、production では常に全エントリーを処理する。

**3. デフォルトで distDir を完全削除**  
`clean: true`（デフォルト）により、ビルド開始前に出力ディレクトリを削除してから再生成する。前回ビルドの残骸が混入しないようにする。

**4. 依存グラフも build では使わない**  
`imageGraph`（Pug→画像依存）は dev 専用とし、production コードパスには含めない。`sassGraph`・`scriptGraph` は build 中に再構築するが、インクリメンタル絞り込みには使わない。

---

## 設計の意図

- **シンプルさ優先** — build は「毎回同じ結果を出す」ことを保証する。キャッシュによる予期しない挙動を排除する
- **予期せぬ変更漏れを防ぐ** — dev でも保存したファイルは必ず書き出す。コンテンツ比較によるスキップは行わない
- **既存環境への組み込みに対応** — htdocs 等の既存ディレクトリに pugkit の生成ファイルを追加・更新するユースケースを想定している
